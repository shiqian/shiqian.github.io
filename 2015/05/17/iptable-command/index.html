structure of iptables

iptables contain multiple tables: filter table, nat table, mangle table and raw table.
filter is default table for iptables. filter table has the following built-in chains:

* INPUT chain - incoming to firewall. for packets coming to the local server.
* OUTPUT chain - outgoing from firewall. for packets generated locally and going out of the local server.
* FORWARD chain -packet for another NIC on the local server. for packets routed through the local server.

example

For example, hosts located in the local network are authorized to established outbound connections with hosts in the internet for a set of application protocols identified by port number {Pi} and to transfer data as part of these connections. The ruleset to implement this policy would be as follows:

``` bash
iptables -F FORWARD
iptables -N FORWARD
iptables -P FORWARD DROP
iptables -A FOREARD -I eth1 -s A.B.C/24 -dport Pi -m state --state NEW,ESTABLISHED -j ACCEPT
iptables -A FOREARD -I eth2 -d A.B.C/24 -sport Pi -m state --state ESTABLISHED -j ACCEPT
```

This solution relies on the stateful inspection feature.
